Epic EHR Integration Reporting System

üìç Project Context

This project was developed at Quiktek to modernize and optimize how patient admission and lab result data was extracted, processed, and delivered. The legacy process took over 6 hours to generate reports, causing bottlenecks in care coordination and data access.

After implementation, the total report turnaround time was reduced to ~6 minutes, enabling near-real-time data access from Epic EHR across reporting endpoints.

  • Integrates directly with Epic EHR using secure REST APIs (FHIR/HL7 standards)

  • Feeds data into SSIS for transformation and stores in a report DB
  • Generates formatted reports using SSRS
  • Automates secure file transfer to reporting consumers via SFTP
  • Monitors system health using Prometheus, Grafana, and Splunk

  • Docker + Kubernetes (EKS on AWS): Containerized and scaled each microservice and reporting component

  • SSIS: Transformed raw patient data into clean, structured datasets
  • SSRS: Rendered reports as PDFs/CSV for downstream delivery
  • Wireshark: Inspected packet-level issues with secure APIs
  • Splunk: Logged and monitored for HIPAA-compliant audit trails
  • Nagios & Prometheus: System-level alerting and monitoring
  • Grafana: Dashboards showing delivery rates, failures, metrics
  • Linux HA Clusters: Redundant systems for reporting uptime
